# Data Model & Firestore Schema

Complete reference for data structures, TypeScript types, and Firestore collections.

---

## 📋 TypeScript Types

All type definitions are in `src/types/index.ts`.

### HabitFrequency

```typescript
type HabitFrequency = 'daily' | 'weekly' | 'monthly'
```

Determines how the habit's heatmap is displayed:
- `daily` — 365-day grid (52 weeks × 7 days)
- `weekly` — 52-cell linear strip (one per week)
- `monthly` — 12-cell linear strip (one per month)

### HabitRating

```typescript
type HabitRating = 1 | 2 | 3 | 4 | 5
```

User's completion level when marking a pixel:
- `1` → Barely completed (20% intensity)
- `2` → Partially completed (40% intensity)
- `3` → Completed (60% intensity, default)
- `4` → Fully completed (80% intensity)
- `5` → Exceeded (100% intensity, darkest color)

Used to determine pixel color via `tierToBackground(rating, habitColor)`.

### HeatmapTier

```typescript
type HeatmapTier = 0 | 1 | 2 | 3 | 4 | 5
```

Visual intensity tier (computed from rating):
- `0` — Empty (no entry for this day/week/month)
- `1-5` — Maps 1:1 from HabitRating

Used internally by heatmap rendering; not user-facing.

### Habit

```typescript
interface Habit {
  id: string                          // Firestore doc ID (auto-generated)
  name: string                        // User-provided, trimmed, max 60 chars
  emoji: string                       // User-provided, max 2 chars
  color: string                       // Hex color (e.g., "#2196f3")
  frequency: HabitFrequency          // 'daily' | 'weekly' | 'monthly'
  createdAt: string                  // ISO timestamp (e.g., "2025-01-15T08:30:00.000Z")
}
```

**Validation Rules**:
- `name` must not be empty after trim
- `name` must be unique (no duplicates)
- `name` max 60 characters
- `emoji` optional, max 2 characters
- `color` must be one of 16 preset hex colors (see `HABIT_COLORS[]` in `colorUtils.ts`)
- `frequency` one of the three values
- `createdAt` auto-generated on creation (not user-settable)

**Example**:
```json
{
  "id": "1abc2def3ghi",
  "name": "Morning Run",
  "emoji": "🏃",
  "color": "#2196f3",
  "frequency": "daily",
  "createdAt": "2025-01-15T08:30:00.000Z"
}
```

### HabitEntry

```typescript
interface HabitEntry {
  id: string                          // Firestore doc ID (auto-generated)
  habitId: string                     // Foreign key → Habit.id
  date: string                        // 'YYYY-MM-DD' format
  rating: HabitRating                // 1 | 2 | 3 | 4 | 5
  description: string                // User notes, max 500 chars
}
```

**Date Logic**:
- `date` stored as string 'YYYY-MM-DD' (not timestamp) for easier grouping/querying
- For daily habits: the literal day (e.g., "2025-01-15")
- For weekly habits: the Monday of that week (e.g., "2025-01-13" for Mon–Sun week)
- For monthly habits: the 1st of that month (e.g., "2025-01-01")

**Uniqueness Constraint**:
- Only one HabitEntry per (habitId, date) pair
- If user marks the same pixel twice, the entry is updated (not duplicated)

**Example**:
```json
{
  "id": "entry-xyz789",
  "habitId": "1abc2def3ghi",
  "date": "2025-01-15",
  "rating": 4,
  "description": "Ran 5 miles in 45 minutes, felt great!"
}
```

### HeatmapCell

```typescript
interface HeatmapCell {
  date: Date                         // JavaScript Date object
  dateStr: string                    // 'YYYY-MM-DD' format
  tier: HeatmapTier                 // 0 | 1 | 2 | 3 | 4 | 5
  entry: HabitEntry | null          // Linked entry (null if empty)
  label: string                      // Display text (e.g., "Jan 15, 2025")
  isFuture: boolean                 // Whether date > today
}
```

**Computed On-the-fly** — not stored in Firestore. Generated by `useHeatmap()` composable:
- One HeatmapCell per pixel in the grid
- `tier` computed from `entry.rating` if entry exists, else `0`
- `isFuture` always `false` (app hides future pixels)
- `label` formatted for tooltip display

**Example**:
```javascript
{
  date: Date("2025-01-15"),
  dateStr: "2025-01-15",
  tier: 4,
  entry: { id: "...", habitId: "...", date: "2025-01-15", rating: 4, description: "..." },
  label: "Wednesday, January 15, 2025",
  isFuture: false
}
```

## 🗄️ Firestore Collections

Pixel Habits uses Firestore (Google Cloud Firestore) as its backend database. All data is stored under per-user namespaces for privacy — users can only read/write their own data via Firebase Authentication security rules.

### Collection: `users/{userId}/habits`

**Document ID**: Firestore auto-generates

**Document Structure**:
```javascript
{
  name: "Morning Run",
  emoji: "🏃",
  color: "#2196f3",
  frequency: "daily",
  createdAt: "2025-01-15T08:30:00.000Z"
}
```

**Security**: Only the owner (authenticated user with matching `uid`) can read/write

**Typical Operations**:
- `addDoc(collection(db, 'users', userId, 'habits'), {...})` — Create
- `updateDoc(doc(db, 'users', userId, 'habits', id), {...})` — Update
- `deleteDoc(doc(db, 'users', userId, 'habits', id))` — Delete (+ batch delete all associated entries)
- `onSnapshot(collection(db, 'users', userId, 'habits'), callback)` — Real-time sync

### Collection: `users/{userId}/entries`

**Document ID**: Firestore auto-generated

**Document Structure**:
```javascript
{
  habitId: "1abc2def3ghi",
  date: "2025-01-15",
  rating: 4,
  description: "Ran 5 miles in 45 minutes"
}
```

**Security**: Only the owner (authenticated user with matching `uid`) can read/write

**Typical Operations**:
- `addDoc(collection(db, 'users', userId, 'entries'), {...})` — Create entry
- `updateDoc(doc(db, 'users', userId, 'entries', id), {...})` — Update rating/description
- `deleteDoc(doc(db, 'users', userId, 'entries', id))` — Delete entry
- `onSnapshot(collection(db, 'users', userId, 'entries'), callback)` — Real-time sync

## 🔄 Data Relationships

```
┌─────────────┐        1:N        ┌──────────────┐
│   Habit     │◄─────────────────►│ HabitEntry   │
│ (template)  │     (entries)     │  (ratings)   │
└─────────────┘                   └──────────────┘
     ↓                                   ↓
  id = PK                          habitId = FK → Habit.id
  name                             date
  emoji                            rating
  color                            description
  frequency
  createdAt
```

**Cascade Delete**: When a habit is deleted, all associated entries must be deleted (app handles this via batched write).

## 📅 Date Handling

### Storage Format: ISO 8601 for `createdAt`, YYYY-MM-DD for `date`

**Timestamps** (when habit is created):
```javascript
habit.createdAt = "2025-01-15T08:30:00.000Z"  // ISO 8601
```
Generated via `new Date().toISOString()`.

**Dates** (when pixels are marked):
```javascript
entry.date = "2025-01-15"  // YYYY-MM-DD, not timestamp
```

Why YYYY-MM-DD?
- Easier to group/query by date
- No time-of-day ambiguity (time zones)
- Natural for heatmap rendering (one cell per day)
- Simple string comparison (`"2025-01-15" > "2025-01-14"`)

### Date Calculation Examples

**Today's date**:
```typescript
import { format } from 'date-fns'
const todayStr = format(new Date(), 'yyyy-MM-dd')  // "2025-01-15"
```

**Parse date string back to Date**:
```typescript
import { parseISO } from 'date-fns'
const dateObj = parseISO("2025-01-15")  // Date object
```

**Week start (for weekly habit)**:
```typescript
import { startOfWeek } from 'date-fns'
const monday = startOfWeek(new Date(), { weekStartsOn: 1 })
const mondayStr = format(monday, 'yyyy-MM-dd')  // e.g., "2025-01-13"
```

**Month start (for monthly habit)**:
```typescript
import { startOfMonth } from 'date-fns'
const firstOfMonth = startOfMonth(new Date())
const firstStr = format(firstOfMonth, 'yyyy-MM-dd')  // e.g., "2025-01-01"
```

## 🎨 Color Model

### 16 Preset Colors

All habits must use one of these colors (defined in `src/utils/colorUtils.ts` → `HABIT_COLORS[]`):

```
#ef5350  (red)
#e91e63  (pink)
#9c27b0  (purple)
#673ab7  (deep purple)
#3f51b5  (indigo)
#2196f3  (blue)
#03a9f4  (light blue)
#00bcd4  (cyan)
#009688  (teal)
#4caf50  (green)
#8bc34a  (light green)
#cddc39  (lime)
#ffc107  (amber)
#ff9800  (orange)
#ff5722  (deep orange)
#795548  (brown)
```

### Pixel Color Calculation

Each pixel's color is computed from:
1. Base empty color: `--heatmap-0` (theme-dependent, light gray or dark gray)
2. Habit color: User's selected hex color
3. Rating: 1–5 stars (how complete the task was)

**Formula**:
```css
/* color-mix blends the two colors by percentage */
backgroundColor: color-mix(in srgb, emptyColor N%, habitColor)
```

Where `N = (5 - rating) * 20`:
- Rating 5 → N = 0% → 0% empty, 100% habit → darkest (full color)
- Rating 4 → N = 20% → 20% empty, 80% habit
- Rating 3 → N = 40% → 40% empty, 60% habit → default
- Rating 2 → N = 60% → 60% empty, 40% habit
- Rating 1 → N = 80% → 80% empty, 20% habit → lightest

**Visual Gradient** (for green habit #4caf50):
```
Rating 1: light pale green (mostly background showing through)
Rating 2: pale green
Rating 3: medium green (default)
Rating 4: dark green
Rating 5: very dark green (full saturation)
```

**CSS Variable System**:
```scss
// Light theme
--heatmap-0: #ebedf0;    // Empty cell (light gray)
--heatmap-1: #9be9a8;    // Deprecated (not used anymore)

// Dark theme
--heatmap-0: #21262d;    // Empty cell (dark gray)
```

## 🧮 Computed Fields

These are **NOT stored** — computed on-the-fly from habits and entries:

### Per-Habit Streaks

**Current Streak** (e.g., for daily habit):
- Count backwards from today through consecutive days with entries
- Stop at first gap
- Return count

**Longest Streak**:
- Scan all entries (sorted by date)
- Find the longest gap-free sequence
- Return count

### Today's Progress (Daily Habits Only)

**todayCompleted**:
```typescript
dailyHabits.filter(h => 
  entries.some(e => e.habitId === h.id && e.date === todayStr)
).length
```

**todayTotal**:
```typescript
dailyHabits.length
```

**todayPercent**:
```typescript
(todayCompleted / todayTotal) * 100
```

## 📊 Data Consistency

### Invariants (must always be true)

1. **Every Habit must have unique ID**
   - Firestore auto-generates, so always true

2. **Every Habit must have unique name**
   - App enforces at creation + edit
   - If violated, edit dialog rejects

3. **Every HabitEntry must have valid habitId**
   - App only creates entries via store
   - App deletes orphaned entries on habit delete (batched)

4. **At most one HabitEntry per (habitId, date)**
   - App checks on `setEntry()` before inserting
   - Updates if exists, adds if new

5. **Dates in HabitEntry are valid YYYY-MM-DD**
   - App generates dates via `format(date, 'yyyy-MM-dd')` from date-fns
   - No manual date strings

### Atomic Operations

**Habit Deletion**:
```typescript
const batch = writeBatch(db)
batch.delete(doc(db, 'habits', id))
for (const entry of entries where habitId === id) {
  batch.delete(doc(db, 'entries', entry.id))
}
await batch.commit()  // All-or-nothing
```

Ensures no orphaned entries if process crashes.
